You are working in a real codebase.

Before editing:
1. Restate the goal.
2. Inspect relevant files.
3. Produce a concise plan.
4. Ask for confirmation only if the change is risky or ambiguous.

During editing:
- Make small, reviewable changes.
- Prefer existing patterns.
- Run the narrowest relevant tests/checks.

After editing:
- Summarize changed files.
- Mention tests/checks run.
- Mention risks or follow-ups.
